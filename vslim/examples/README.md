# VSlim Examples

This directory contains runnable example apps for `vslim`.

## hello-app.php

`hello-app.php` is the current minimal end-to-end example. It demonstrates:

- `VSlimApp`
- `before()` / `after()`
- route params
- route groups
- named routes
- `url_for()` / `redirect_to()`

```php
<?php
declare(strict_types=1);

$app = new VSlimApp();

$app->get_named('hello.show', '/hello/:name', function (VSlimRequest $req) {
    return new VSlimResponse(
        200,
        'Hello, ' . $req->param('name'),
        'text/plain; charset=utf-8'
    );
});

$app->get('/go/:name', function (VSlimRequest $req) use ($app) {
    return $app->redirect_to('hello.show', ['name' => $req->param('name')]);
});
```

## Build

From the `vslim` directory:

```bash
cd /Users/guweigang/Source/vphpext/vslim
make build
make vhttpd
```

## Run

Start `vhttpd` with the example app bootstrap:

```bash
VSLIM_HTTPD_APP=/Users/guweigang/Source/vphpext/vslim/examples/hello-app.php \
make -C /Users/guweigang/Source/vphpext/vslim serve
```

If you want a different port:

```bash
VSLIM_HTTPD_APP=/Users/guweigang/Source/vphpext/vslim/examples/hello-app.php \
VHTTPD_PORT=19884 \
make -C /Users/guweigang/Source/vphpext/vslim serve
```

## Try it

Use `--noproxy '*'` for local smoke tests so shell/system proxies do not hijack
the request:

```bash
curl --noproxy '*' -i http://127.0.0.1:19881/hello/codex
curl --noproxy '*' -i http://127.0.0.1:19881/go/nova
curl --noproxy '*' -i -H 'Host: demo.local' http://127.0.0.1:19881/api/meta
```

Expected behavior:

- `/hello/codex`
  - returns `200 OK`
  - body: `Hello, codex`

- `/go/nova`
  - returns a redirect to `/hello/nova`

- `/api/meta`
  - returns JSON metadata generated by the example app

## Notes

- `vhttpd` is the HTTP/runtime layer and is built on `veb`
- `php-worker.php` is the worker boundary
- `hello-app.php` returns a `VSlimApp`, which the worker dispatches directly
- `/dispatch?...` still exists as a debug bridge endpoint, but the example app
  is intended to be used through real HTTP paths such as `/hello/codex`
